# Phase 5: GitHub Actions Pipelines

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Three working GitHub Actions workflows: CI on every push/PR, a Changesets-driven release workflow that publishes both npm packages + pushes the Docker image to GHCR, and a Docker build+smoke-test workflow on PRs that touch the Dockerfile or packages.

**Architecture:** All workflows use `ubuntu-latest`. Turborepo remote cache via `TURBO_TOKEN`. Release is tag-triggered by Changesets merging a "Version PR" into `main`. Docker image is pushed to `ghcr.io/${{ github.repository_owner }}/druckform`.

**Tech Stack:** GitHub Actions, Turborepo remote cache, Changesets GitHub bot, GHCR, npm provenance attestations.

## Global Constraints

- CI must pass before any merge to `main`
- Coverage gate (80% line) enforced in CI via Vitest
- npm publish uses `--provenance` for supply-chain attestations
- Docker image tagged: `latest`, semver (e.g. `1.2.3`), minor alias (e.g. `1.2`)
- `TURBO_TOKEN` and `TURBO_TEAM` secrets for remote cache
- `NODE_AUTH_TOKEN` secret for npm publish
- `GITHUB_TOKEN` used for GHCR (no extra secret needed)
- HTTP server binds to `127.0.0.1` — do not publish port `-p` in Docker run commands

### Task 1: CI workflow

**Files:**
- Create: `.github/workflows/ci.yml`

- [ ] **Step 1: Write `.github/workflows/ci.yml`**

```yaml
name: CI

on:
  push:
    branches: [main]
  pull_request:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  ci:
    name: Typecheck / Lint / Build / Test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9.14.0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Typecheck
        run: pnpm turbo typecheck
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}

      - name: Lint
        run: pnpm turbo lint
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}

      - name: Build
        run: pnpm turbo build
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}

      - name: Test (with coverage)
        run: pnpm turbo test
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}

      - name: Upload coverage
        uses: actions/upload-artifact@v4
        if: always()
        with:
          name: coverage
          path: |
            packages/druckform/coverage/
            packages/druckform-mcp/coverage/
          retention-days: 7
```

### Task 2: Release workflow

**Files:**
- Create: `.github/workflows/release.yml`

- [ ] **Step 1: Write `.github/workflows/release.yml`**

```yaml
name: Release

on:
  push:
    branches: [main]

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  release:
    name: Changesets → npm + GHCR
    runs-on: ubuntu-latest
    permissions:
      contents: write       # create release commits + tags
      pull-requests: write  # open/update Version PRs
      packages: write       # push to GHCR
      id-token: write       # npm provenance attestations

    steps:
      - name: Checkout
        uses: actions/checkout@v4
        with:
          # Full history needed for Changesets to determine what changed
          fetch-depth: 0

      - name: Setup pnpm
        uses: pnpm/action-setup@v4
        with:
          version: 9.14.0

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 22
          cache: pnpm
          registry-url: https://registry.npmjs.org

      - name: Install dependencies
        run: pnpm install --frozen-lockfile

      - name: Build packages
        run: pnpm turbo build
        env:
          TURBO_TOKEN: ${{ secrets.TURBO_TOKEN }}
          TURBO_TEAM: ${{ secrets.TURBO_TEAM }}

      - name: Changesets — open/update Version PR or publish
        id: changesets
        uses: changesets/action@v1
        with:
          publish: pnpm release
          title: "chore: version packages"
          commit: "chore: version packages"
        env:
          GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
          NODE_AUTH_TOKEN: ${{ secrets.NODE_AUTH_TOKEN }}
          NPM_CONFIG_PROVENANCE: "true"

      # Only push Docker image when packages were actually published
      - name: Log in to GHCR
        if: steps.changesets.outputs.published == 'true'
        uses: docker/login-action@v3
        with:
          registry: ghcr.io
          username: ${{ github.repository_owner }}
          password: ${{ secrets.GITHUB_TOKEN }}

      - name: Extract version from druckform package
        if: steps.changesets.outputs.published == 'true'
        id: version
        run: |
          VERSION=$(node -p "require('./packages/druckform/package.json').version")
          MINOR="${VERSION%.*}"
          echo "version=$VERSION" >> "$GITHUB_OUTPUT"
          echo "minor=$MINOR" >> "$GITHUB_OUTPUT"

      - name: Set up Docker Buildx
        if: steps.changesets.outputs.published == 'true'
        uses: docker/setup-buildx-action@v3

      - name: Build and push Docker image
        if: steps.changesets.outputs.published == 'true'
        uses: docker/build-push-action@v6
        with:
          context: .
          push: true
          tags: |
            ghcr.io/${{ github.repository_owner }}/druckform:latest
            ghcr.io/${{ github.repository_owner }}/druckform:${{ steps.version.outputs.version }}
            ghcr.io/${{ github.repository_owner }}/druckform:${{ steps.version.outputs.minor }}
          cache-from: type=gha
          cache-to: type=gha,mode=max
          provenance: true
          sbom: true
```

### Task 3: Docker build+test workflow

**Files:**
- Create: `.github/workflows/docker-build-test.yml`

- [ ] **Step 1: Write `.github/workflows/docker-build-test.yml`**

```yaml
name: Docker Build & Smoke Test

on:
  pull_request:
    branches: [main]
    paths:
      - Dockerfile
      - .dockerignore
      - docker/**
      - packages/**
      - pnpm-lock.yaml

concurrency:
  group: ${{ github.workflow }}-${{ github.ref }}
  cancel-in-progress: true

jobs:
  docker-smoke:
    name: Build image + smoke test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Set up Docker Buildx
        uses: docker/setup-buildx-action@v3

      - name: Build Docker image (no push)
        uses: docker/build-push-action@v6
        with:
          context: .
          push: false
          load: true
          tags: druckform:ci-test
          cache-from: type=gha
          cache-to: type=gha,mode=max

      - name: Smoke — druck --help exits 0
        run: docker run --rm druckform:ci-test --help

      - name: Smoke — druck templates --json is valid contract
        run: |
          docker run --rm druckform:ci-test templates --json > /tmp/templates.json
          node -e "
            const d = require('/tmp/templates.json');
            if (d.schemaVersion !== '1') throw new Error('bad schemaVersion');
            if (!Array.isArray(d.templates)) throw new Error('no templates array');
            console.log('OK - templates contract valid, count:', d.templates.length);
          "

      - name: Smoke — druck components --template base --json is valid contract
        run: |
          docker run --rm druckform:ci-test components --template base --json > /tmp/components.json
          node -e "
            const d = require('/tmp/components.json');
            if (d.schemaVersion !== '1') throw new Error('bad schemaVersion');
            if (!d.components.some(c => c.name === 'infobox')) throw new Error('infobox missing');
            console.log('OK - components contract valid');
          "

      - name: Smoke — druck lint (valid fixture) exits 0
        run: |
          docker run --rm \
            -v "${{ github.workspace }}/packages/druckform/tests/fixtures:/fixtures:ro" \
            druckform:ci-test lint \
              --template base \
              --in /fixtures/documents/valid.md \
              --json > /tmp/lint.json
          node -e "
            const d = require('/tmp/lint.json');
            if (!d.ok) throw new Error('lint failed: ' + JSON.stringify(d.findings));
            console.log('OK - lint passed');
          "

      - name: Smoke — druck lint (invalid fixture) exits non-zero
        run: |
          set +e
          docker run --rm \
            -v "${{ github.workspace }}/packages/druckform/tests/fixtures:/fixtures:ro" \
            druckform:ci-test lint \
              --template base \
              --in /fixtures/documents/invalid-missing-required.md \
              --json > /tmp/lint-invalid.json
          EXIT_CODE=$?
          set -e
          node -e "
            const d = require('/tmp/lint-invalid.json');
            if (d.ok) throw new Error('Expected lint to fail but it passed');
            if (d.findings.length === 0) throw new Error('Expected findings but got none');
            console.log('OK - invalid document correctly rejected');
          "
          if [ "$EXIT_CODE" -eq 0 ]; then
            echo "ERROR: Expected non-zero exit code for invalid document"
            exit 1
          fi
```

# GitHub Actions — Required Secrets

| Secret | Used by | How to obtain |
|---|---|---|
| `TURBO_TOKEN` | ci.yml, release.yml | Turborepo remote cache — create at [turbo.build](https://turbo.build/repo/docs/core-concepts/remote-caching) or leave unset to skip caching |

| `NODE_AUTH_TOKEN` | release.yml | npm access token with `publish` scope — create at npmjs.com → Access Tokens |
